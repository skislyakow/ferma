# Telegram Channel Automation — Brain Dump & Checklist

## 1. Философия: что продаём?

Telegram-канал — это **трафик-актив**. Люди приходят за контентом, а мы конвертируем их в деньги.
Канал не обязан быть "живым" — он обязан быть **полезным/интересным** и **регулярным**.

Три слоя монетизации:
- **Прямая реклама** — посты с рекламой за деньги (Telegram Ads, биржи, прямые контракты)
- **Партнёрский маркетинг (CPA)** — referral-ссылки в контент
- **Продажа канала** — вырастил → продал на бирже (Telega.io, etc.)

## 3. Архитектура автоматизации

```
[Парсер] → [База контента] → [Фильтр/Дедупликация] → [Перевод (если надо)] → [Пост-прокладка] → [Публикация по расписанию]
```

### 3.1 Парсер
- **Telegram API (MTProto)** через Telethon или pyrogram
- Парсим **контент + engagement-метрики** (просмотры, реакции)
- Цель: собирать **лучшие посты** из каналов-доноров
- Критерий отбора: топ-30% по просмотрам/реакциям

### 3.2 Перевод
- **DeepL API** или **GPT-4o-mini** (контекстный перевод, адаптация под tone of voice)
- Если канал-донор англоязычный → перевод на русский
- Если донор русский → можно оставить как есть или перевести на английский для global audience

### 3.3 Пост-прокладка
- Добавление CPA-ссылок / реферальных кодов в контент
- Рерайт заголовка для виральности
- Добавление CTA (Call to Action)

### 3.4 Публикация
- Расписание: равномерно, 3-5 постов в день
- Использовать Telethon для публикации
- Вариант: Telegram Bot API (но есть лимиты)

## 4. Бизнес-модель (MVP)

### Вариант A: "Парсинг + Перевод + Реклама"
1. Находим топ-3 донора в нише crypto/AI
2. Парсим их топ-посты
3. Переводим (если иноязычные)
4. Добавляем свою CPA-ссылку (например, BingX, ByBit, Airdrop-проекты)
5. Публикуем 3-5 раз/день
6. Через 2-3 месяца канал набирает 1000+ подписчиков → подключаем Telegram Ads
7. **Profit:** $200-1000/мес с одного канала

### Вариант B: "Сетка каналов"
- 5-10 каналов в разных нишах
- Один парсер → несколько каналов
- Автоматизация через единый скрипт
- **Profit:** $1000-5000/мес

### Вариант C: "Buy & Flip"
- Покупаем каналы на Telega.io / Telderi
- Накручиваем контент + подписчиков
- Продаём через 2-3 месяца
- **Profit:** 30-100% за сделку

## 5. Чек-лист реализации

### Этап 1: Инфраструктура
- [ ] Выбрать нишу (рекомендую начать с крипты/AI)
- [ ] Найти 3-5 каналов-доноров с высокой активностью
- [ ] Получить API ID/Hash Telegram (my.telegram.org)
- [ ] Поднять сервер/RDP/VPS (рекомендую Linux VPS за $5-10/мес)
- [ ] Установить Python 3.11+, настроить virtualenv

### Этап 2: Парсер
- [ ] Написать скрипт парсинга (Telethon)
- [ ] Собирать: текст, медиа, просмотры, реакции, дата
- [ ] Сохранять в SQLite/PostgreSQL
- [ ] Фильтр: оставлять только посты с engagement > median
- [ ] Дедупликация (чтобы не постить одинаковое)

### Этап 3: Перевод
- [ ] Подключить DeepL API или OpenAI API
- [ ] Написать модуль перевода с сохранением форматирования Markdown/HTML
- [ ] Тест: проверка качества перевода на тестовой выборке

### Этап 4: Генерация поста
- [ ] Шаблонизатор: добавлять заголовок, CTA, хэштеги
- [ ] Вставка CPA-ссылок (менеджер ссылок с ротацией)
- [ ] Прокладка рекламы: каждый N-ый пост — рекламный

### Этап 5: Публикация
- [ ] Скрипт публикации в канал
- [ ] Планировщик (schedule) — cron / APScheduler
- [ ] Расписание: пост утром, днём, вечером

### Этап 6: Монетизация
- [ ] Подключить Telegram Ads (нужно 1000+ подписчиков)
- [ ] Найти прямых рекламодателей (биржи: Telega.io, AdsGram, Serpstat)
- [ ] Вставить CPA-ссылки в посты (реферальные программы криптобирж)
- [ ] Мониторинг CTR и дохода

### Этап 7: Масштабирование
- [ ] Запустить 2-3 канала параллельно
- [ ] A/B тест ниш
- [ ] Автоматизировать закупку подписчиков (если нужно для минимального порога)
- [ ] Дойти до $1000/мес, потом реинвестировать

## 10. Ключевой принцип

> Не делай контент. Управляй потоком контента.

Мы не создаём — мы **курируем с умом**. Парсинг, фильтрация, адаптация, публикация. Полный цикл в автоматическом режиме. Вмешательство — раз в неделю на 30 минут (проверить метрики, сменить доноров).

**С 1 канала:** $200-1000/мес
**С 5 каналов:** $1000-5000/мес
**С 10 каналов:** $3000-10000/мес

Работает чисто на объёме и автоматизации.

## 11. Фактическая реализация (MVP — 31.05.2026)

### Итоговая архитектура:
```
t.me/s/ (веб-парсинг) → Yandex Translate → Telegram Bot API
```

### Почему не оригинальный план:
| Планировалось | По факту | Причина |
|--------------|----------|---------|
| Telethon (MTProto) | WebParser (t.me/s/) | my.telegram.org не дал API ключи |
| DeepL API | Yandex Translate API | DeepL Free закрыт для РФ |
| Telethon (публикация) | Bot API (HTTP) | Bot API не требует API ID/Hash |

### Вывод:
Через Bot API + t.me/s/ можно полноценно автоматизировать канал **без единого API-ключа Telegram**. Для РФ это оптимальный путь, т.к. my.telegram.org часто недоступен.

- Ссылка вставляется автоматически каждый N-ый пост

**Фильтры (core/filter/filters.py):**
- Блокировка рекламы: fragment.com, retweet, presale, десант и др. (≥2 ключевых слов)
- Внешние источники: "читайте на", "подробнее на", "на сайте", "источник:" и др.
- Дедупликация: MD5 первых 200 символов
- Пропущенные посты: помечаются published=-1, не перепроверяются
- Ротация: 1 пост на донора (PARTITION BY source_channel)

**Фильтры контента (core/publisher/publisher.py):**
- Очистка футеров: "мы в", "подпишись", "присоединяйся", URL-ссылки
- Если текст > 1024 символов — картинка пропускается (sendMessage вместо sendPhoto)

**Известные баги (починены):**
- `hash()` → `int(msg_id)` — message ID теперь стабильный между запусками (раньше hash() рандомизировался PYTHONHASHSEED, дублируя посты)
- `caption` → `text` — несуществующая переменная в sendPhoto
- `mark_skipped()` — отфильтрованные посты не перепроверяются
- `try/except` в цикле — процесс не умирает от SSL-ошибок Telegram API

**VPS**: Timeweb (Нидерланды) — ожидает деплоя (без VPS t.me нестабилен из РФ)
